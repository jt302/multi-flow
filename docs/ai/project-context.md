# 项目背景（project-context）

## 1. 项目一句话

用 Tauri + React 构建一款桌面“多环境浏览器管理器”，以自研/魔改 Chromium 144 为引擎，实现：
多账号环境隔离 + 指纹与网络配置 + 扩展治理 + 自动化（窗口同步 / Local API / MCP）+ 团队协作治理。

## 2. 为什么要做

1. 业务侧痛点

- 多账号运营需要环境隔离：cookie/缓存/本地存储/扩展等必须互不污染
- 代理与环境绑定、批量运维困难：需要代理资产池 + 检测 + 批量分配
- 重复操作多：窗口同步与可扩展自动化接口能显著提效
- 团队协作风险：账号共享、密码散落、权限粗放、日志不可追溯

2. 技术侧机会

- 已有自研/魔改 Chromium 144，可以做深度一致性的指纹控制与启动编排
- 用 Tauri 将“系统能力（进程/文件/加密/权限）”与“UI 管理台”分离
- 可按 AdsPower 的“Local API + MCP server”生态路线构建可扩展自动化接口

## 3. 对标功能拆解（以 AdsPower 为参照）

> 下面是“能力面清单”，用于定义我们要做的模块边界与验收口径。
> 注意：AdsPower 功能会随版本迭代变化，本清单用于“对标与需求归档”。

A. 环境/配置文件（Profile）

- 列表管理：搜索、筛选、显示列管理、排序
- 创建：基础信息（名称/分组/标签/备注）、账号平台、cookie 导入、指纹配置、高级设置
- 打开/关闭：批量打开、批量关闭、打开状态展示、最后打开时间
- 导入/导出：
  - Excel / txt 导出（txt 主要解决 cookie 超长问题）
  - 批量创建（Excel 模板 / txt 模板）
- 批量操作：
  - 批量更新环境信息（名称/账号/分辨率/代理等）
  - 批量更改 UA/内核
  - 批量修改指纹项（时区/WebRTC/地理位置/语言/Do Not Track/端口扫描保护等）
  - 批量清理缓存（localStorage/indexedDB/cookie/插件数据/历史等）
- 分享：
  - 可撤回/不可撤回
  - 选择分享信息（名称/代理/备注/标签页…）
  - 指定接收者（邮箱/手机号）

B. 分组管理（Group）

- 创建/修改/备注
- 导出、合并、分享、置顶
- 授权子用户管理分组
- 排序规则清晰、支持大量分组

C. 回收站（Recycle Bin）

- 软删除环境进入回收站
- 30 天内可筛选与恢复
- 恢复不超过套餐可用环境数
- 支持彻底删除（不可恢复）

D. 代理管理（Proxy Pool）

- 支持协议：HTTP/HTTPS/SSH/SOCKS5
- 批量添加（多行）、检查代理、标签与备注
- 关联环境数、到期时间、供应商字段
- 创建环境时可引用代理池代理（或随机分配）
- 可选：集成动态代理类型与密钥同步

E. 应用中心（Extensions / App Center）

- 团队应用：统一启用/禁用、按环境跟随安装
- 推荐应用：一键添加常用扩展
- 上传扩展：zip/crx 解包上传、图标/名称/简介
- 分类管理
- 安全策略：扩展不自动更新（可选）

F. 窗口同步（Synchronizer）

- 主控 -> 被控：鼠标、键盘、标签页、文本输入
- 操作台：窗口管理 / 文本管理 / 标签页管理
- 排列方式：宫格平铺、重叠平铺、自定义排列
- 桌面小组件（可选）
- 高级：跨屏同步、子窗口同步、独立剪贴板（可选）

G. API & MCP（自动化生态）

- Local API：profile/group/proxy/extensions 等 CRUD，启动/关闭引擎
- API Key 安全校验 + 访问频率限制 + 调用日志
- 返回调试信息（CDP/WebDriver）供第三方自动化框架使用
- MCP server：把 Local API 封装为 MCP tools，供 AI/Agent 调用

H. 团队协作治理

- 成员管理（用户组、权限分配、环境/分组授权、席位计费）
- 操作日志（登录/IP/打开/环境变更）
- 全局设置（启动策略、代理失败策略、禁止加载图片/视频、数据同步策略、缓存目录策略）
- 费用中心（如商业化）：套餐、续期、统计、余额/赠送金

I. 数据安全与同步

- 本地优先存储
- 可选：端到端加密云同步（按数据类型可控）
- 支持彻底清除本地与云端数据
- 安全锁/互斥打开（可选）

## 4. 产品边界（Non-Goals）

- 不承诺绕过任何平台风控或用于违规行为
- 不做“账号买卖/黑灰产”相关辅助
- 首期不做复杂的云端计费与大规模 SaaS（除非你明确要商业化）

## 5. 里程碑（建议）

M0（跑起来）：工程脚手架 + 基础导航 + Profile CRUD（本地）
M1（可用）：引擎启动/关闭 + 代理池 + cookie 导入 + 导出
M2（提效）：窗口同步 v1 + Local API v1
M3（可扩展）：应用中心 v1
M4（团队化）：成员/权限/日志/全局设置
M5（生态）：MCP server / 模板市场（按需）
